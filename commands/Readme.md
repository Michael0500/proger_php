# XBRL-CSV конвертер для шаблона ЦБ "МР УОД ПУРЦБ"

Yii2 консольная команда. Конвертирует xlsx-шаблон Банка России
("МР УОД ПУРЦБ 6.1.0.7 (финальная)") в пакет XBRL-CSV для сдачи в ЦБ
по точке входа `ep_nso_purcb_oper_nr_uod_reestr` (таксономия 6.1.0.7
от 2025-07-04, схема DRAFT 2025-07-31).

Реализовано строго по «Правилам формирования УОД ПУРЦБ в формате XBRL-CSV»
(версия 2.0 от 01.08.2025), сверено бит-в-бит с эталоном `example_full.zip`.

## Что делает

1. Читает лист **`Прил 1`** xlsx-шаблона — это полный реестр concept'ов таксономии
   (раздел / QName / itemType / instant|duration / префикс / roleUri).
2. Идёт по листам **`Раздел 1.` … `Раздел 11.`**:
   - строка 4 — заголовки с QName в скобках `(dim-int:C_CdTaxis)`, `(purcb-dic:Cntrct_BDt)` и т.п.;
   - строка 5 — служебная (Open Open) — пропускается;
   - со строки 6 — данные.
3. На каждый непустой раздел пишет `sr_R{N}.csv`. Пустые разделы в пакет не включаются
   (по §2.3.2 Правил), но отмечаются в `sr_sved_purcb.csv` как
   `mem-int:DannyeOtsutstvuyutMember`. Заполненные разделы получают флаг
   `mem-int:DannyePeredanyMember`.
4. Генерит `mapping.json` в полном виде по §2.2.7.
5. Складывает всё в `.zip` без вложенных папок (по §2.1).

## Установка

```bash
composer require phpoffice/phpspreadsheet
```

`XbrlConvertController.php` положить в `console/controllers/`.

## Запуск

Положите xlsx-шаблон в **`web/uploads/report.xlsx`** (относительно корня проекта).
Готовый пакет появится в **`web/xbrl-output/`**.

```bash
php yii xbrl-convert/run \
    --identifier=1027700000000 \
    --report-date=2025-10-31 \
    --period-start=2025-08-01 \
    --period-end=2025-10-31 \
    --request-number=00012_38
```

Пути захардкожены в константах в начале контроллера:
```php
private const INPUT_RELATIVE  = 'web/uploads/report.xlsx';
private const OUTPUT_RELATIVE = 'web/xbrl-output';
```
Если нужно изменить — поправьте константы.

### Параметры

| Опция | Обязателен | Описание |
|---|---|---|
| `--identifier` | да | ОГРН отчитывающейся организации (13 цифр) |
| `--identifier-predecessor` | нет | ОГРН правопредшественника (для реорганизаций, §3) |
| `--report-date` | да | дата среза `YYYY-MM-DD` (поле `reportDate`) |
| `--period-start` | нет | начало отчётного периода — кладётся в `sr_sved_purcb.csv` |
| `--period-end` | нет | конец отчётного периода — кладётся в `sr_sved_purcb.csv` |
| `--request-number` | нет | номер запроса в формате `XXXXX_XX` (default `00000_00`) |
| `--delimiter` | нет | `\|` (default) или `,` — §2.3.6 рекомендует `\|` |
| `--zip` | нет | `1` (default) — упаковать в zip |

### Что получится на выходе

```
1027700000000_ep_nso_purcb_oper_nr_uod_reestr_00012_38_20251031/
├── mapping.json
├── sr_sved_purcb.csv
├── sr_R1.csv         ← только если Раздел 1 непустой
├── sr_R2.csv
└── …

CSV_1027700000000_ep_nso_purcb_oper_nr_uod_reestr_00012_38_20251031.zip   ← готовый архив 1 уровня
```

После этого вам остаётся (по §2.1, шаги 2–5):

1. Заархивировать полученный `.zip` ещё раз во **внешний** zip
   (имя без префикса `CSV_`).
2. Подписать его УКЭП → файл `.zip.sig` (КриптоАрм ГОСТ).
3. Зашифровать → `.zip.enc`.
4. Загрузить `.sig` и `.enc` в Личный кабинет ЕПВВ.

Эти шаги — снаружи команды (требуют ваш сертификат).

## Известные ограничения

1. **xlsb на входе.** PhpSpreadsheet не читает xlsb. Конвертация:
   ```bash
   soffice --headless --convert-to xlsx /path/to/file.xlsb --outdir /tmp
   ```
   Или через Excel: Save As → Excel Workbook (.xlsx).

2. **`typedDomain`.** Для типизированных осей соглашение замены `Taxis → TypedName`
   работает не всегда (например `dim-int_Cntrct_CdTaxis → dim-int_ID_Dogovora_Typedname`).
   В команде захардкожен полный словарь из 16 пар осей таксономии 6.1.0.7.
   При обновлении версии таксономии — пересобрать константу `TYPED_DOMAINS`
   из официального `example_full/mapping.json`.

3. **Контрольные соотношения (Прил 3).** В пакет не кладутся — это проверяется
   на стороне ЦБ при приёме. Если КС не выполняется — по §2.5 рекомендуется
   приложить `pz.doc` с пояснениями.

4. **`sr_sved_purcb.csv` упрощённый.** В команде заполняется только базовый
   набор: `PeriodStart_Dt`, `PeriodEnd_Dt` и 11 флагов R{N}Enumerator.
   В реальном пакете там 22 поля (PSMP_INN, PSMP_Nm, OtvZaPrInf_FIO/Tlf/ElPoch,
   Inf_PO, PR7Enumerator, Rqst_Dt/Rspns_Dt и др.) — их сейчас можно
   дозаполнить параметрами командной строки или прямо в коде в
   `buildSvedTable()`. Если нужно — допишу поддержку всех полей через
   yaml-конфиг или дополнительные опции `--operator-fio`, `--operator-phone` и т.п.

## Если что-то пошло не так

Перед сдачей прогоните пакет через **Arelle** или официальный валидатор
ЦБ: он покажет конкретные расхождения с DRAFT-схемой. Скиньте мне вывод —
доточим.
